## 2. 레슨 대상자(골퍼) 분석

### 2-1. 연령별 골프 인구

| 연령대 | 비중 (2024) | 트렌드 | 특성 |
|--------|------------|--------|------|
| **20대** | 4.8% | 급감 ↓↓ | 유행 소비, 이탈 중 |
| **30대** | 18.1% | 감소 ↓ | 골린이 세대, 일부 이탈 |
| **40대** | **53%** | **주도층** ↑ | 핵심 소비층, 안정적 |
| **50대** | 14.5% | 안정 → | 충성도 높음 |
| **60대+** | 9.6% | 미세 증가 ↑ | 건강/사교 목적 |

### 2-2. 골퍼 유형별 분석

#### A. "골린이" (초보 골퍼) - 축소 중
- **정의**: 골프 입문 1~2년차
- **연령**: 주로 30~40대
- **동기**: 사회적 네트워킹, 취미, 접대
- **지출**: 월 30~40만 원 (타석 + 레슨)
- **Pain Point**: 비용 부담, 실력 정체, 레슨 선택 어려움
- **현황**: MZ세대 이탈로 신규 유입 둔화

#### B. "실력 향상파" (중급 골퍼) - 핵심 타겟
- **정의**: 골프 3년 이상, 90대~100대 타수
- **연령**: 주로 40~50대
- **동기**: 스코어 개선, 안정적 스윙
- **지출**: 월 50~100만 원 (레슨 + 라운드)
- **Pain Point**: 실력 정체, 체계적 코칭 부재
- **현황**: 가장 안정적인 레슨 수요층

#### C. "프리미엄 골퍼" (상급) - 고단가
- **정의**: 싱글 핸디캡 또는 목표
- **연령**: 40~60대
- **동기**: 경기력 극대화
- **지출**: 월 100만 원+ (프리미엄 1:1 레슨)
- **Pain Point**: 데이터 기반 분석 요구, 좋은 프로 찾기 어려움
- **현황**: 소수지만 높은 지불 의사

#### D. "주니어/여성" - 성장 중
- **정의**: 10대 선수 지망생, 여성 입문자
- **연령**: 10대, 30~40대 여성
- **동기**: 선수 육성, 사교/건강
- **지출**: 월 50~200만 원 (주니어), 30~50만 원 (여성)
- **현황**: 골프장/아카데미의 전략적 육성 대상

### 2-3. 레슨 비용 시세 (2025)

| 레슨 유형 | 월 비용 | 비고 |
|-----------|---------|------|
| 실내 그룹 레슨 | 15만 원 | 주 2~3회, 10~15분씩 |
| 실내 개인 레슨 | 30만 원 | 주 2회, 30분씩 |

**수강권 구조:**
- 1개월 8회 / 2개월 16회 / 3개월 24회가 표준

## 3. 레슨 프로 분석

### 3-1. 레슨 프로 규모 (추정)

| 자격 단체 | 회원/자격자 수 | 비고 |
|----------|---------------|------|
| KPGA | 6,468+ 명 (2018) | 매년 50명+ 신규 투어프로 |
| KLPGA | 수천 명 (추정) | 티칭프로~마스터프로 |
| KSPGA | 수천 명 | 세미프로/티칭프로 특화 |
| USGTF-Korea | 수천 명 | 240만원이면 취득 가능 |
| KGTA | 수백~수천 명 | 대한골프지도자협회 |
| 기타 (10개+ 단체) | 수천 명 | 민간 자격증 |
| **합계 (추정)** | **20,000~30,000명** | 활동 중인 프로 기준 |

> 법적으로 골프 레슨에 자격증은 필수가 아님 → 실제 레슨 활동자는 더 많을 수 있음

### 3-2. 프로 유형별 분석

#### 유형 1: 연습장 소속 프로 (가장 다수)
```
수입 구조: 기본급 120~200만 + 레슨비 배분 (50~70%)
월 수입: 250~500만 원
회원 수: 15~30명
근무 형태: 주 6일, 오전~저녁
Pain Point:
  - 연습장 수수료가 높다 (30~50%)
  - 이직 잦음 → 고객 데이터 유실
  - 카카오톡으로 예약 관리 → 혼선
  - "알바생 프로" 인식
```

#### 유형 2: 프리랜서/개인 프로
```
수입 구조: 레슨비 100% - 시설 임대료
월 수입: 300~800만 원 (편차 큼)
회원 수: 10~40명
근무 형태: 자유, 여러 곳 순회
Pain Point:
  - 고객 확보를 직접 해야 함 (마케팅 부담)
  - 스케줄 관리 복잡 (여러 시설)
  - 결제/수금 관리 어려움
  - 세금 신고/수입 추적 불투명
```

#### 유형 3: 아카데미 운영 프로
```
수입 구조: 레슨비 + 시설 운영 수익
월 수입: 500~2,000만 원
회원 수: 30~100+ 명
직원: 1~5명 프로 고용
Pain Point:
  - 강사 관리/급여 계산
  - 시설 운영 + 레슨 병행 부담
  - 체계적 회원 관리 시스템 부재
  - 경쟁 아카데미와 차별화
```

#### 유형 4: 스타/유튜버 프로
```
수입 구조: 레슨비 + 광고/협찬 + 콘텐츠
월 수입: 1,000만 원+
회원 수: 소수 프리미엄 (대기 리스트)
Pain Point:
  - 시간 관리 (콘텐츠 + 레슨)
  - 프리미엄 고객 기대 관리
  - 브랜드 관리
```

### 3-3. 프로의 핵심 Pain Points (종합)

| 순위 | Pain Point | 심각도 | 현재 해결책 |
|------|-----------|--------|------------|
| **1** | **예약/스케줄 관리** | ★★★★★ | 카카오톡, 수기 메모 |
| **2** | **고객 이력 관리** | ★★★★★ | 기억, 메모장 |
| **3** | **결제/수금 추적** | ★★★★☆ | 계좌이체, 현금, 엑셀 |
| **4** | **고객 유지/리텐션** | ★★★★☆ | 감에 의존 |
| **5** | **신규 고객 확보** | ★★★★☆ | 입소문, 김캐디, 인스타 |
| **6** | **레슨 품질 차별화** | ★★★☆☆ | 개인 노하우 |
| **7** | **수입 관리/세금** | ★★★☆☆ | 엑셀, 세무사 |
| **8** | **온라인/비대면 레슨** | ★★☆☆☆ | 거의 못 함 |

### 3-4. 프로의 고객 관리 현실

```
현재 워크플로우 (대다수 프로):

1. 신규 고객 → 카카오톡 친구 추가
2. 레슨 상담 → 카카오톡 채팅
3. 스케줄 확정 → 카카오톡 + 머릿속 기억
4. 결제 → 계좌이체 or 현금 (매번 확인)
5. 레슨 진행 → 구두 피드백
6. 레슨 기록 → 없음 (기억에 의존)
7. 패키지 만료 → 직접 연락 (까먹으면 이탈)
8. 실력 추적 → 없음
9. 수입 관리 → 통장 내역 or 엑셀

→ "하루에 평균 5건 이상 오던 수강 문의가 1주에 1~2건 정도로 줄었다"
→ 시장 축소 + 비체계적 관리 = 수입 감소
```

## 4. 기존 경쟁 서비스 분석 (한국)

### 4-1. 레슨북 (LessonBook)

**포지션: "센터 운영 관리 도구"**

| 항목 | 내용 |
|------|------|
| 출시 | 2015년 5월 |
| 누적 다운로드 | 50만+ |
| 대상 업종 | 골프, 필라테스, PT, 피아노, 과외 등 전 분야 |
| 개발사 | 포인트뱅크 (판교) |
| 앱 가격 | 무료 (상세 요금제 비공개) |

**강점 (잘하는 것):**
- 예약/스케줄 관리 (월간/주간/일간)
- 회원 예약/취소 (회원이 직접)
- 레슨 횟수/기간 자동 카운트
- 센터 현황 대시보드 (이용중/만기/소진/중지)
- 입출금 관리
- 강사 배정 + 출결 관리 + 자동 수당 계산
- 간편결제 (수강권 앱 결제)
- 골프 특화: 타석예약, 키오스크, 무인운영, QR 출입

**약점 (못하는 것):**
- ❌ 영상 분석 기능 없음
- ❌ AI 기능 없음
- ❌ 비동기 영상 레슨/피드백 불가
- ❌ 학생 성장 추적/리포트 없음
- ❌ 자동화/템플릿 없음
- ❌ 참여도 분석(Engagement Insights) 없음
- ❌ 골프 특화 코칭 도구 부재 (범용 레슨 관리)

**사용자 후기:**
- "이 앱 없으면 너무 혼란스러울 것 같아요" (필라테스 강사)
- "고객들한테 메시지도 가고 자동 출석체크가 돼서 편합니다" (골프 프로)
- "원비 수납이나 레슨 횟수도 자동 정리" (긍정)
- "듀엣수업 등록 시 불편" (개선 요청)

### 4-2. 김캐디 (KimCaddie)

**포지션: "골퍼 중심 예약/비교 플랫폼"**

| 항목 | 내용 |
|------|------|
| 다운로드 | 200만+ |
| 투자 유치 | 100억 원+ |
| 레슨 매칭률 | 약 30% |
| 확장 | 미국/일본 진출 본격화 |

**레슨 관련 기능:**
- 프로 프로필 열람 (약력, 전문 분야)
- 가격 비교 (1회/3개월/6개월)
- 맞춤 레슨 요청서
- 1:1 비대면 상담
- AI 스윙 분석
- AI 스코어 기록

**포인트:** 김캐디는 "골퍼(수요자)"를 위한 서비스. 프로를 위한 CRM/관리 도구가 아님.

### 4-3. 스마트스코어

**포지션: "골퍼 종합 플랫폼"**

- 360만+ 사용자
- 스코어 관리 유료화 → 사용자 불만 급증
- 레슨 기능: 연습장 프로 후기 정도 (부가 기능)
- 프로 CRM 기능: 없음

### 4-4. 경쟁 공백 분석

```
                    [골퍼(B2C)]              [프로(B2B)]
                        │                        │
    매칭/예약 ─────── 김캐디 ◄──────────────► (없음)
                    스마트스코어
                        │                        │
    운영/관리 ────────── (없음) ◄──────────────► 레슨북 ← 범용(골프 특화 아님)
                        │                        │
    코칭/분석 ────────── (없음) ◄──────────────► (없음) ← ★ 여기가 기회
                        │                        │
                    [글로벌만]               [글로벌만]
                     Skillest              CoachNow, V1
```

**핵심 발견:**
> "프로를 위한 골프 특화 코칭 도구"는 한국에 **진짜 없다**
>
> - 레슨북 = 범용 레슨 관리 (골프/필라/PT 공통) → "운영"에 집중
> - 김캐디 = 골퍼용 매칭 → 프로 관리 도구 아님
> - CoachNow의 "코칭 품질 향상" 영역이 한국에서는 비어 있음

## 7. 타겟 고객 세분화

### 7-1. 1차 타겟: 프리랜서/개인 레슨 프로

| 항목 | 내용 |
|------|------|
| **규모** | 추정 10,000~15,000명 |
| **나이** | 25~45세 |
| **월 수입** | 300~800만 원 |
| **회원 수** | 10~40명 |
| **디지털 활용** | 중~상 (인스타, 유튜브 활용) |
| **지불 의사** | 월 3~5만 원 (추정, 검증 필요) |
| **핵심 니즈** | 고객 관리 체계화, 신규 유입, 수입 안정화 |

**왜 1차 타겟?**
- 레슨북의 핵심 타겟은 "센터"이지만, 개인 프로에 대한 지원은 약함
- 프리랜서 프로는 "카카오톡 + 수기"에 의존 → 전환 가치가 높음
- 자기 브랜딩에 관심 → AI 리포트 등 프리미엄 기능에 반응할 가능성

### 7-2. 2차 타겟: 소규모 아카데미 (프로 1~3명)

| 항목 | 내용 |
|------|------|
| **규모** | 추정 3,000~5,000개 |
| **월 수입** | 500~2,000만 원 |
| **회원 수** | 30~100명 |
| **핵심 니즈** | 센터 운영 + 코칭 품질 동시 관리 |
| **지불 의사** | 월 5~10만 원 |

### 7-3. 3차 타겟: 연습장 소속 프로

- 레슨북/연습장 자체 시스템에 이미 종속
- 전환 비용이 높아 후순위

## 8. 시장 기회 재평가

### 8-1. TAM/SAM/SOM (수정)

| 단계 | 계산 | 금액 |
|------|------|------|
| **TAM** | 한국 레슨 프로 30,000명 × 월 5만 원 × 12 | **180억 원/년** |
| **SAM** | 디지털 활용 프로 10,000명 × 월 5만 원 × 12 | **60억 원/년** |
| **SOM (2년)** | 프리랜서 프로 1,000명 × 월 4만 원 × 12 | **4.8억 원/년** |

### 8-2. 현실적 리스크

| 리스크 | 심각도 | 대응 |
|--------|--------|------|
| 레슨북이 코칭 기능 추가 | ★★★★☆ | 선점 + AI 차별화 |
| 김캐디가 프로 CRM 진출 | ★★★★☆ | 니치 타겟 (개인 프로) 집중 |
| CoachNow 한국 진출 | ★★★☆☆ | 한국 특화 (카카오톡, 문화) |
| 프로의 지불 의사 부족 | ★★★★★ | **반드시 인터뷰로 검증** |
| 골프 시장 지속 축소 | ★★★☆☆ | 4050 핵심층은 안정적 |

### 8-3. Go/No-Go 판단 기준

**Go 조건 (3개 이상 충족 시):**
- [ ] 인터뷰 프로 10명 중 7명+ 이 "이런 서비스 필요하다" 응답
- [ ] 월 3~5만 원 지불 의사 50%+ 확인
- [ ] 레슨북 사용자의 불만 포인트 3개+ 발견
- [ ] 레슨북 미사용 프로의 이유 확인 ("필요 없어서" vs "몰라서" vs "불편해서")
- [ ] 프로 5명+ 이 "영상 피드백/AI 리포트" 기능에 강한 관심

**No-Go 조건:**
- "카카오톡이면 충분하다"가 다수 의견
- 레슨북으로 이미 만족하는 프로가 대다수
- 월 2만 원도 아까워하는 반응

## 11. Golfearn 레슨 프로 CRM 개발 계획

### 11-1. 프로젝트 개요

**서비스명:** Golfearn Pro CRM
**타겟:** 한국 골프 레슨 프로 (프리랜서/개인 중심)
**핵심 가치:** "카카오톡 대체 + AI 코칭 도구"

### 11-2. 재사용 가능한 기존 자산

| 기존 자산 | 활용 방안 |
|-----------|----------|
| Supabase Auth | 프로/학생 로그인 |
| Next.js 14 + TypeScript | 프레임워크 그대로 사용 |
| Tailwind CSS | UI 스타일링 |
| UI 컴포넌트 (Button, Card, Input) | 기본 UI 요소 |
| Lemon Squeezy 결제 | 프로 구독 결제 |
| Claude API | AI 레슨 노트 생성 |
| 다국어 시스템 (next-intl) | 추후 글로벌 확장 |

### 11-3. 새로운 DB 스키마

```sql
-- 1. 레슨 프로 프로필 확장
ALTER TABLE profiles ADD COLUMN IF NOT EXISTS
  is_lesson_pro BOOLEAN DEFAULT false,
  pro_certification VARCHAR(200),
  pro_experience_years INTEGER,
  pro_specialties TEXT[],
  pro_introduction TEXT,
  pro_monthly_fee INTEGER;

-- 2. 학생 관리
CREATE TABLE lesson_students (
  id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
  pro_id UUID REFERENCES profiles(id) ON DELETE CASCADE,
  student_name VARCHAR(100) NOT NULL,
  student_phone VARCHAR(20),
  student_email VARCHAR(100),
  student_memo TEXT,
  current_level VARCHAR(50),
  goal TEXT,
  created_at TIMESTAMPTZ DEFAULT NOW(),
  last_lesson_at TIMESTAMPTZ,
  total_lesson_count INTEGER DEFAULT 0,
  is_active BOOLEAN DEFAULT true
);

-- 3. 수강권/패키지
CREATE TABLE lesson_packages (
  id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
  pro_id UUID REFERENCES profiles(id) ON DELETE CASCADE,
  student_id UUID REFERENCES lesson_students(id) ON DELETE CASCADE,
  package_name VARCHAR(100) NOT NULL,
  total_count INTEGER NOT NULL,
  used_count INTEGER DEFAULT 0,
  price INTEGER NOT NULL,
  start_date DATE NOT NULL,
  end_date DATE NOT NULL,
  status VARCHAR(20) DEFAULT 'active',
  payment_status VARCHAR(20) DEFAULT 'pending',
  paid_amount INTEGER DEFAULT 0,
  created_at TIMESTAMPTZ DEFAULT NOW()
);

-- 4. 레슨 스케줄
CREATE TABLE lesson_schedules (
  id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
  pro_id UUID REFERENCES profiles(id) ON DELETE CASCADE,
  student_id UUID REFERENCES lesson_students(id) ON DELETE CASCADE,
  package_id UUID REFERENCES lesson_packages(id),
  lesson_date DATE NOT NULL,
  lesson_time TIME NOT NULL,
  duration_minutes INTEGER DEFAULT 50,
  status VARCHAR(20) DEFAULT 'scheduled',
  location VARCHAR(200),
  memo TEXT,
  created_at TIMESTAMPTZ DEFAULT NOW()
);

-- 5. AI 레슨 노트
CREATE TABLE lesson_notes (
  id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
  schedule_id UUID REFERENCES lesson_schedules(id) ON DELETE CASCADE,
  pro_id UUID REFERENCES profiles(id) ON DELETE CASCADE,
  student_id UUID REFERENCES lesson_students(id) ON DELETE CASCADE,
  voice_transcript TEXT,
  ai_structured_note JSONB,
  key_points TEXT[],
  homework TEXT,
  next_focus TEXT,
  created_at TIMESTAMPTZ DEFAULT NOW()
);

-- 6. 수입 관리
CREATE TABLE pro_income_records (
  id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
  pro_id UUID REFERENCES profiles(id) ON DELETE CASCADE,
  student_id UUID REFERENCES lesson_students(id),
  package_id UUID REFERENCES lesson_packages(id),
  amount INTEGER NOT NULL,
  payment_method VARCHAR(50),
  payment_date DATE NOT NULL,
  memo TEXT,
  created_at TIMESTAMPTZ DEFAULT NOW()
);

-- 인덱스 생성
CREATE INDEX idx_lesson_students_pro ON lesson_students(pro_id);
CREATE INDEX idx_lesson_packages_student ON lesson_packages(student_id);
CREATE INDEX idx_lesson_schedules_date ON lesson_schedules(lesson_date);
CREATE INDEX idx_lesson_schedules_pro ON lesson_schedules(pro_id);
CREATE INDEX idx_pro_income_date ON pro_income_records(payment_date);
```

### 11-4. 개발 로드맵

#### Phase 1: MVP (2주)

// 신규 개발 필요 컴포넌트

  DashboardStats.tsx  // 통계 위젯

  ScheduleCalendar.tsx // 캘린더 컴포넌트

  IncomeChart.tsx     // 수입 차트
  LessonNoteEditor.tsx // AI 노트 에디터

  lesson-students.ts  // 학생 CRUD
